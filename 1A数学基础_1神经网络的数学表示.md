# 神经网络如何用数据表示
### 基本单位 —— 张量(tensor)
>    本质是多维Numpy数组
>    张量是矩阵向任意维度(通常被称为轴，或者阶)的推广
##### 1. 标量
    只有一个float32或float64数字的张量被称为标量或0D张量
    你可以使用ndim属性来检查一个Numpy张量的轴的个数
##### 2. 向量
    由数字组成的数组被称为向量或一维张量
    假设一个有x个元素的张量，就可以被称为xD向量、x阶向量
    这里的D是指该向量的维度，表示某个轴上的元素个数
    而xD张量表示的是该张量的轴，轴上可能有任意维度
##### 3. 矩阵
    向量的数组即可成为矩阵或二维张量，它有两个轴(行、列)
    以下面数据为例：
    [5, 7, 8, 0]为第一行、[6, 3, 1, 8]为第一列
|5|7|8|0|
|----|----|----|----|
|6|3|1|8|
##### 4. 3D张量与更高维张量
>将多个矩阵合成新的数组得到一个3D张量
也就是一个数字组成的立方体，下面是在程序中的表示方法
[ [ [5, 6],
    [2, 3],
    [0, 9] ],
[ [1, 4],
    [7, 3],
    [8, 0] ],
[ [3, 7],
    [5, 0],
    [1, 8] ] ]
##### 5. 张量的关键属性
* 轴的个数(阶)
* 形状 —— 表示张量每个轴的维度大小(标量为空 -> ())
* 数据类型 —— 一般是float32、float64、uint8等
### 张量的操作
和数组的操作类似 `name[x]` 即为选择了第一个轴的数字
也可以称为张量切片（可以使用python的语法
`name[10:100]`就是切出了name下10到100的数字（不包括第100
##### 1. 数据批量
通常来说深度学习中的第一个轴，被称为`样本轴 样本维度`
还有，深度学习的模型一般不会同时处理整个数据集，而是拆成诸多的小批量数据处理
MNIST一个批量为128，这类批量张量也叫`批量轴 批量维度`
##### 2. 现实世界的数据张量
1. 向量数据 2D张量 形状为`(samples, features)`
2. (时间)序列数据 3D张量 形状为`(samples, timesteps, features)`
3. 图像数据 4D张量 形状为`(samples, height, width, channels)`有时最后一轴会变成第二轴
4. 视频数据 5D张量 形状为`(samples, frames, height, width, channels)`有时最后一轴会变成第三轴
##### 3. 向量数据
> 最为常见的数据

每个数据点都被编码为一个向量，所以一个数据批量就被编码成2D张量
第一个轴为样本轴，第二个轴为特征轴
##### 4. (时间)序列数据
每个样本被编码成2D向量，再将样本组成3D张量
按照惯例，时间为第二个轴
##### 5. 图像数据
图像通常具有3个维度：高、宽、颜色深度
由于灰度图像只有一个色彩通道因此可以保存在2D张量中
形状一般为`样本, 高度, 宽度, 色彩通道`或者`样本, 色彩通道, 高度, 宽度`
前者一般在tensorFlow中使用，后者一般在channels中应用
而keras同时支持
##### 6. 视频数据
首先每一帧为`(height, width, color_depth)`的3D张量
一系列帧为`(frames, height, width, color_depth)`的4D张量
而不同视频的批量就是`(samples, frames, height, width, color_depth)`的5D张量
